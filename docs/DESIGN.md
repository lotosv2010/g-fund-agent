# 视觉与交互设计规范（DESIGN）

> 本文档定义产品的视觉语言和交互模式，是 AI 辅助开发的 UI 约束。
> 所有用户可见的输出必须遵循本规范，确保体验一致性。

## 1. 设计原则

| 原则 | 说明 |
|------|------|
| 信息密度优先 | CLI 工具追求高效信息展示，避免冗余装饰 |
| 颜色即语义 | 颜色仅用于传达含义（涨/跌、成功/失败），不做纯装饰 |
| 渐进式展示 | 先概览后详情，避免一次性输出过多信息 |
| 一致性 | 同类信息在不同场景中使用相同的展示格式 |

## 2. 色彩系统

基于 `chalk` 实现，统一由 `src/utils/colors.ts` 导出。

| 语义 | 颜色 | 用途 |
|------|------|------|
| 正向 | `chalk.green` | 上涨、加仓、成功、确认 |
| 负向 | `chalk.red` | 下跌、减仓、错误、警告 |
| 中性提示 | `chalk.cyan` | 加载状态、spinner、进度提示 |
| 辅助信息 | `chalk.gray` / `chalk.dim` | 次要说明、提示文字、分隔线 |
| 警告 | `chalk.yellow` | 输入格式错误、缺失数据、非致命提示 |
| 强调 | `chalk.bold` | 标题、关键数值、汇总行 |

### 涨跌数值格式

```
上涨: chalk.green("+1.23%")    // 正数带 + 号
下跌: chalk.red("-0.45%")      // 负数保持 - 号
```

## 3. 交互模式

### 3.1 导航结构

```
启动 → 选择模型 → 功能菜单（循环）
                      ├── 更新持仓 → Agent 获取数据 → 交互收集操作 → 确认保存
                      ├── 查看持仓 → Agent 回复 → 展示
                      ├── 分析持仓 → Agent 回复 → 展示
                      └── 自由对话 → 对话循环 → exit 退出
```

### 3.2 选择菜单

使用 `@inquirer/prompts` 的 `select` 组件，附带 `chalk.dim` 描述文字：

```
? 选择功能:
❯ 更新持仓 — 拉取涨跌 + 记录操作
  查看持仓 — 查看持仓概览
  分析持仓 — 全面诊断当前持仓
  自由对话 — 基金问答
```

### 3.3 确认操作

所有持久化操作（文件写入）必须经过 `confirm` 确认，默认值根据场景设置：

| 场景 | 默认值 |
|------|--------|
| 保存持仓更新 | `true`（常规操作，减少摩擦） |
| 是否有交易操作 | `false`（多数日期无操作） |

### 3.4 输入收集

交易操作输入采用自由文本模式，格式：`基金代码 金额`

```
操作> 161631 500      → 已记录: 161631 +500
操作> 014320 -200     → 已记录: 014320 -200
操作> done             → 结束输入
```

输入校验即时反馈（`chalk.yellow`）：
- 格式错误 → 提示正确格式
- 基金代码不存在 → 提示可用代码
- 金额非法 → 提示要求

## 4. 信息展示

### 4.1 加载状态

使用 Braille 字符动画 spinner + `chalk.cyan`：

```
⠋ 思考中...
⠙ 正在连接 MCP 服务...
```

完成后清除 spinner 行，不留痕迹。

### 4.2 持仓变更对比

核心输出格式，每行包含：基金代码 + 名称（截断 14 字符）+ 金额变化 + 涨跌 + 操作 + 累计收益率

```
持仓变更对比:

  161631 富国沪深300增强    12000.00 → 12147.60  +1.23% (+147.60)  累计 +5.32%
  014320 德邦半导体         8000.00 → 7820.00   -2.25% (-180.00) [加仓 +500]  累计 -3.12%

  合计  20000.00 → 19967.60  -32.40  操作 +500.00
```

**规则：**
- 金额保留 2 位小数
- 百分比保留 2 位小数，带 `%` 后缀
- 加仓/减仓标签仅在有操作时显示
- 汇总行始终显示

### 4.3 Agent 回复

Agent 返回的 Markdown 直接输出到终端，前后各空一行：

```
\n{Agent Markdown 回复}\n
```

无回复时显示 `chalk.yellow("（无回复）")`。

### 4.4 错误展示

```
错误: {错误信息}              → chalk.red("错误:") + 消息
致命错误: {错误信息}           → chalk.red("致命错误:") + 消息，随后 process.exit(1)
```

### 4.5 状态信息

```
使用模型: deepseek              → chalk.green + chalk.bold
Agent 就绪。可用工具: 12 个     → chalk.green + chalk.gray
数据来源: data/portfolio-...    → chalk.gray
已获取 10 只基金的涨跌数据      → chalk.green
```

## 5. 文本规范

| 类别 | 规则 |
|------|------|
| 语言 | 所有面向用户的文案使用中文 |
| 标点 | 中文标点（逗号、句号、冒号） |
| 退出提示 | "输入 exit 退出" |
| 确认提示 | "确认保存更新？" 简短直接 |
| 空状态 | 明确告知原因（如"加载持仓失败"），不使用空白或沉默 |

## 6. 响应式适配

当前仅支持终端输出，遵循以下原则：

- 基金名称截断至 14 字符，避免表格溢出
- 不假设终端宽度，避免硬编码列宽
- Spinner 动画清除时覆盖整行，不留残影
