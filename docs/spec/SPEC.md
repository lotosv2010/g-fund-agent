# G-Fund-Agent 需求规格说明书

## 1. 概述

基金持仓自动分析 Agent。根据持仓基金的市场表现，按预设规则判断是否需要调仓，并输出调仓建议。

**核心原则：纯规则驱动，禁止主观判断。**

## 2. 功能需求

1. 每周周日晚 10 点手动触发，Agent 分析持仓基金是否需要调仓
2. 触发规则时，输出具体调仓建议（买入/卖出哪只、金额多少）
3. 未触发任何规则时，输出"本周无需调仓"并记录持仓快照
4. 每次运行必须输出净值检查表，无论是否触发操作

## 3. 目标仓位

总目标资金：**200,000 元**

| 资产类别 | 目标占比 | 目标金额 |
|----------|----------|----------|
| 宽基类 | 35% | 70,000 元 |
| 科技主题类 | 15% | 30,000 元 |
| 海外类 | 15% | 30,000 元 |
| 债券 | 30% | 60,000 元 |
| 黄金 | 5% | 10,000 元 |

## 4. 当前持仓

详见 [portfolio.md](../portfolio.md)，该文件为持仓数据的唯一数据源，每周更新。

## 5. 调仓规则

### 关键概念

- **近期高点**：该基金过去 60 个交易日内的最高净值
- **补仓点**：每次执行补仓操作时记录的净值，作为后续止盈的基准
- **档位**：补仓/止盈分多档执行，各档独立触发、累进操作
- **目标仓位比例**：操作金额以该基金在目标组合中的金额为基准计算

## 6. 技术架构

### 6.1 技术栈

- Runtime: Node.js + TypeScript + pnpm
- AI 框架: deepagents + langchain + @langchain/langgraph
- CLI: langgraphjs (dev/build/up/down)
- LLM: Ollama (本地模型)
- 数据源: qieman MCP (且慢)
- 追踪: LangSmith

### 6.2 Agent 架构

```
createDeepAgent (主编排器)
├── data_fetcher (子 Agent: LLM + qieman MCP Tools)
├── reporter     (子 Agent: LLM + save_report Tool)
└── 纯计算函数
    ├── calculateMarket()
    ├── matchRules()
    └── optimizePortfolio()
```

- **子 Agent**：使用 LLM 调用工具（MCP 数据获取 / 文件写入），不做决策
- **纯计算函数**：无 LLM，纯规则/计算逻辑，确保所有买卖决策可审计

## 7. 强制约束

1. **纯规则驱动** - 所有买卖决策必须由规则触发，禁止 LLM 主观判断
2. **补仓点结构化存储** - 止盈规则依赖补仓点，必须持久化记录每次补仓的净值和时间
3. **总仓位校验** - 每次输出前校验各类资产占比，偏离目标超过阈值时给出校正建议
4. **净值检查必输出** - 每次运行必须输出净值检查表，无论是否触发操作
