# 产品规格说明（SPEC）

> 本文档定义产品的"做什么"和"为谁做"，是 AI 辅助开发的需求边界。
> 所有功能开发必须可追溯到本文档中的场景或需求项。

## 1. 产品概述

| 项目 | 说明 |
|------|------|
| 产品名称 | 顾投 AI 助手（G-Fund-Agent） |
| 目标用户 | 个人基金投资者（有一定投资经验，自主管理组合） |
| 核心价值 | 通过自然语言交互实现持仓分析、基金诊断和组合管理，降低投资决策成本 |
| 一句话定位 | 面向个人投资者的基金顾投 AI 助手 |

## 2. 用户场景

### US-01: 查看持仓

**作为** 投资者，**我想** 查看当前持仓概况，**以便** 快速了解各基金表现。

**验收标准：**
- 展示各基金基本信息 + 最新净值 + 近期收益
- 以表格形式结构化展示
- 数据来源为且慢 MCP 实时获取

### US-02: 分析持仓

**作为** 投资者，**我想** 对持仓进行全面诊断，**以便** 发现潜在风险和优化空间。

**验收标准：**
1. 持仓概览 — 各基金基本信息 + 最新净值 + 近期收益
2. 资产配置 — 穿透后的股/债/现金比例
3. 行业分布 — 前 5 大行业集中度
4. 相关性分析 — 基金间重合度（分散是否充分）
5. 风险评估 — 组合整体风险等级
6. 诊断建议 — 集中度、风格偏移等问题提示
7. 策略评估 — 结合操作策略，判断是否触发补仓/止盈/纪律预警

**数据依赖：** 且慢 MCP（DiagnoseFundPortfolio / GetFundAssetClassAnalysis / GetFundsCorrelation / AnalyzePortfolioRisk 等）

### US-03: 更新持仓

**作为** 投资者，**我想** 更新持仓数据（涨跌 + 交易操作），**以便** 保持持仓记录准确。

**验收标准：**
1. 自动加载最新持仓文件（`data/portfolio-YYYY-MM-DD.json`）
2. 通过 Agent 调用 MCP 获取最近交易日涨跌幅
3. 自动计算新金额和收益率
4. 支持加仓/减仓操作记录
5. 按交易日期保存（同日多次更新覆盖，历史文件不受影响）

### US-04: 自由对话

**作为** 投资者，**我想** 以自然语言咨询基金相关问题，**以便** 获得即时的投资参考信息。

**验收标准：**
- Agent 根据 System Prompt 自主选择工具回答
- 支持基金查询、业绩比较、市场资讯等

## 3. 功能矩阵

| 模块 | 能力 | 优先级 | 状态 |
|------|------|--------|------|
| 持仓分析 | 资产配置、行业集中度、相关性、风险评估、综合诊断 | P0 | ✅ 已完成 |
| 持仓更新 | 涨跌更新、加减仓记录、日期命名文件 | P0 | ✅ 已完成 |
| 基金查询 | 按代码/名称查找，查看详情和业绩 | P0 | ✅ 已完成 |
| 操作策略 | 策略 JSON 注入，分析时自动评估 | P1 | ✅ 已完成 |
| 市场洞察 | 行情解读、热点分析 | P2 | 🔜 规划中 |
| 策略回测 | 定投模拟、组合回测 | P2 | 🔜 规划中 |

## 4. 非功能需求

| 类别 | 要求 |
|------|------|
| 数据来源 | 统一走且慢 MCP，不自建爬虫 |
| 多模型支持 | 用户可选择 DeepSeek / Gemini / Moonshot / MiniMax / Ollama |
| 可观测性 | LangSmith Tracing 全链路追踪 |
| 运行模式 | CLI 交互模式 + LangGraph Server 双模式 |
| 数据持久化 | 本地 JSON 文件（按交易日期命名），无数据库依赖 |

## 5. 约束与边界

**不做：**
- 实时交易执行
- 非基金品种（股票/债券/期货直接交易）
- 用户账户体系
- 移动端 App
- 自建数据爬虫

**免责声明：** 所有分析和建议仅供参考，不构成投资建议。

## 6. 术语表

| 术语 | 说明 |
|------|------|
