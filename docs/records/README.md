# 操作记录

此目录用于存放实际的基金操作记录。

## 文件命名规范

每次操作创建一个文件：`YYYY-MM-DD.md`

示例：
- `2026-03-30.md` - 2026年3月30日的操作
- `2026-04-06.md` - 2026年4月6日的操作

## 记录模板

```markdown
# 操作记录 - YYYY-MM-DD

## 操作类型
- [x] 补仓
- [ ] 止盈
- [ ] 调仓

## 操作明细

### 1. 中欧中证机器人指数C（020256）
- 操作类型：买入
- 金额：3,000 元
- 净值：1.4286
- 触发原因：较近期高点下跌 8.3%，触发第1档补仓
- 备注：无

### 2. 华泰柏瑞纯债（000188）
- 操作类型：卖出
- 金额：2,000 元
- 净值：1.1520
- 触发原因：债券占比超标，减仓平衡
- 备注：无

## 操作后状态

- 总持仓：45,167 元
- 到位率：22.6%
- 本周变化：+1,000 元

## 补仓点记录

如果是补仓操作，记得执行以下命令：

\`\`\`bash
node -e "require('./dist/state/store').addBuyPoint({
  code: '020256',
  date: '2026-03-30',
  nav: 1.4286,
  amount: 3000
})"
\`\`\`

执行后验证：
\`\`\`bash
cat data/buy-points.json
\`\`\`
```

## 注意事项

1. **每次操作后立即记录**，不要等到周末
2. **补仓必须记录补仓点**，否则止盈规则无法触发
3. **保留完整记录**，便于事后复盘和税务处理
4. **金额精确到小数**，与实际交易一致
