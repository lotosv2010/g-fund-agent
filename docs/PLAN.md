# G-Fund-Agent 开发规划

## 产品定位

面向个人投资者的智能量化交易助手。通过 LLM Agent 实现策略开发、回测验证、信号生成和自动化执行的闭环。

## 演进路线

```
Phase 0 (当前)        Phase 1           Phase 2           Phase 3          Phase 4
稳定 V1              数据+风控          策略抽象层         回测+自优化       执行自动化
─────────────────────────────────────────────────────────────────────────────────────
固定规则引擎    →   多源数据+风控层  →  自然语言→策略  →  历史回测验证  →  交易API对接
单一MCP数据     →   宏观/行业/情绪   →  策略注册表     →  归因分析      →  订单管理
Markdown报告    →   风险预警         →  LLM信号生成    →  参数自动调优  →  执行确认闭环
手动触发执行    →   定时触发         →  策略版本管理   →  策略对比评分  →  异常自动熔断
```

## 关键设计决策

1. **渐进演进** — 每个 Phase 独立可用，不依赖后续 Phase 才能工作
2. **风控优先** — Phase 1 先补风控，再谈 LLM 信号和自动执行
3. **LLM 角色演进** — 从 "禁止判断" → "辅助判断（风控护栏内）" → "策略建议（人工确认）"
4. **人在回路** — 即使到 Phase 4，大额操作仍需人工确认，熔断机制不可覆盖
5. **补仓金额基于类别目标** — `类别目标金额 × 档位百分比`（如宽基第 1 档 = 200,000 × 35% × 5% = 3,500 元）
